Building dos9

To get Dos9 binaries, you have two options :

* Download precompiled binaries package from Dos9's sourceforge page. Then you just have to unpack the package on your computer. This method is only availiable for windows. Indeed, for *nix systems, it about going mad to make distribution-portable binaries, for *nix users, the second solution must be used.

* Download sources and then build them yourself. To eases this work, just download sources package from Dos9's sourceforge page, or just get them from the git repository using :

git clone ssh://darkbatcher@git.code.sf.net/p/dos9/code dos9-code

When you have downloaded sources, you just need to compile them using any C compiler. For windows system, it is more complicated than getting precompiled binaries because you need to have gcc and gmake installed on your system. For *nix system, doing so is as simple as typing :

export TARGET_OS=linux make -B

Windows users just have to remove the first line, obviously. Windows users also need to move non native dependencies *.dll from the folder ``depend'' to the folder ``bin''.

First steps

Once you have binaries, you can run Dos9 since it has no need to be installed in order to work. If you want to associate it with .bat or .cmd files though, do not associate it with both file types, as Dos9 is still an unstable software.

Once you launched Dos9, you can type batch commands lines. Remember that if you need some help, the help program is HLP. Note that if you can not browse the documentation due to the lack of translation or else, you can still read the different translations of the documentation in several languages and formats in the subfolder ``share/doc''.

If the executable crash, please send a mail to : <darkbatcher@dos9.org>.

Changing the language

If the language of the interpretor or of the documentation might not be in your langage under MS-WINDOWS, thus you should configure the ``Dos9_Auto.bat'' script that Dos9 launches at startup. Modify the following line :

SET LANGUAGE=en_US

Replace en_US by the name of your local. It is still possible that the langage does not work though, this means that Dos9 is not translated yet in your language. Note that dos9 might be translated into your language, and manual page can be not translated, so if you use HLP, it might crash. Nevertheless, this should'nt occur as HLP failsback to english.

Making translation

Translations uses the GNU gettext tool. To make translation, run the ``dos9-build-gettext.bat'' script to produce .po files for your language, and then, translate it via a po editor like poedit or kbabel.

To submit translation, send it to : <darkbatcher@dos9.org>
